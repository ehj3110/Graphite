# Conformal Lattice Generation Engine: Project Specification

## 1. Project Objective
Build a Python-based, simulation-free lattice generator optimized for resin 3D printing and fracture mechanics research. The tool must intake a boundary representation (STL), fill the volume with a user-specified conformal lattice topology, and iteratively achieve a targeted solid fraction ($V_f$). The output must be a single, watertight, strictly manifold STL ready for slicing, avoiding the severe performance bottlenecks of traditional CAD Boolean operations.

## 2. Geometric Terminology & Topologies
This tool relies entirely on Conformal Meshing, not voxel grids. 

* Vertex-to-Centroid (BCC Tet): Connects the centroid of a tetrahedron to its 4 vertices. High stiffness, stretching-dominated.
* Octet-Truss: Generated by connecting the 6 midpoints of a tetrahedron's edges.
* True 3D Kagome: A 6-connected network forming alternating triangles and hexagonal windows.
* Diamond (Dual Lattice): Connects the centroids of adjacent tetrahedrons across shared faces. Creates 4-connected nodes and 12-sided voids. Excellent isotropic behavior and resin drainage.

## 3. The Technical Stack (Strict Constraints)
* trimesh: For STL import, watertight validation, volume calculation, surface edge extraction, and final export.
* gmsh (via API): The core scaffolding engine. Generates irregular, space-filling conformal tetrahedral meshes.
* numpy & scipy.spatial: For all coordinate mapping and KD-Tree strut deduplication. Vectorized operations only.
* manifold3d (Python Bindings): The geometry kernel. Used exclusively for cylinder generation and hyper-fast batch_union() operations.

## 4. Modular Software Architecture

### Module 1: io_module.py
Logic: Load mesh, repair normals, verify is_watertight, calculate bounding box and total volume.

### Module 2: scaffold_module.py
Logic: Interface with gmsh. Set element size. Generate volumetric mesh. Extract node coordinates, element connectivity lists, and surface face connectivity.

### Module 3: topology_module.py (The Core Engine)
Step A: Iterate through elements to calculate internal local coordinates (e.g., centroids, midpoints).
Step B: Generate internal line segments (Start XYZ to End XYZ).
Step C (Surface Wireframe Cage): Extract the edges of the boundary surface faces and add them to the line segment list to create a clean, conformal outer bounding skeleton. No hanging struts.
Step D (Deduplicate): Use a Hash Map or KD-Tree to ensure shared edges only generate one strut.

### Module 4: geometry_module.py
Step A: Generate a manifold3d.Cylinder for every segment.
Step B (Optional Fillet): Allow a toggle to generate a manifold3d.Sphere (radius slightly larger than cylinder) at every node to relieve stress concentrations at strut intersections.
Step C: Execute manifold3d.batch_union().
Step D (Boundary Trim): Execute a final boolean intersection against the original STL to cleanly slice off any microscopic geometry poking through the boundary.

### Module 5: solver.py
Logic: Target Solid Fraction optimization loop using a Binary Search or Newton-Raphson approach.

## 6. Strict Anti-Patterns (DO NOT DO THESE)
* **NO Dense Voxel Grids:** Do not use `numpy.meshgrid` or 3D voxel arrays to generate the explicit strut lattices. Rely entirely on the boundary-conforming tetrahedrons from `gmsh`.
* **NO Python `for` loops for geometric math:** Do not iterate over 50,000 struts using native loops. You must use vectorized `numpy` broadcasting or `scipy.spatial` structures (like cKDTree) for distance calculations and deduplication.
* **NO Heavy CAD Kernels:** Do not import `bpy` (Blender), `FreeCAD`, or `Open3D`. Stick strictly to `trimesh` and `manifold3d`.
* **NO Monolithic Scripts:** Do not write the entire pipeline in one file. Respect the modular architecture above. 
* **NO Sequential Booleans:** Never use a `for` loop to union struts one by one. You must gather all geometry and use `manifold3d.batch_union()`.
